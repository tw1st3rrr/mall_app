Сделай следующее:

2. Найди все места в HTML/CSS где стоят placeholder-изображения, сломанные пути, заглушки типа `placeholder.jpg`, пустые `src=""`, или комментарии `<!-- TODO: image -->`

4. Подключи фотографии — замени заглушки на реальные пути
5. Для каждого `<img>` убедись что заполнен атрибут `alt` с осмысленным описанием

## Задача 3 — Плавный скролл к форме при клике на мероприятие

На главной странице афиши при клике на карточку мероприятия (или кнопку «Подробнее» / «Записаться» на карточке) — страница должна плавно прокручиваться вниз к форме бронирования.

Реализация:

```js
// Найди все кликабельные элементы карточек мероприятий
// и добавь на них обработчик:

document.querySelectorAll('.event-card, .event-card .btn, .event-card a')
  .forEach(el => {
    el.addEventListener('click', function(e) {
      // Если это не ссылка на отдельную страницу мероприятия — скроллим к форме
      // Если это ссылка на страницу мероприятия — пусть переходит как обычно,
      // но на странице мероприятия тоже добавь плавный скролл к форме если в URL есть #booking

      const form = document.querySelector('#booking, .booking-form, form');
      if (form) {
        e.preventDefault();
        form.scrollIntoView({ behavior: 'smooth', block: 'start' });
      }
    });
  });
```

Уточни селекторы по реальной разметке проекта. Логика:
- Если карточка ведёт на **отдельную страницу мероприятия** — переход происходит как обычно, а на той странице форма уже внизу (ничего дополнительно делать не нужно, пользователь сам доскроллит)
- Если карточка **не ведёт на отдельную страницу** (всё на одной странице) — плавный скролл к форме

Добавь также отступ сверху при скролле чтобы форма не прилипала к самому верху под шапкой:

```css
#booking, .booking-form {
  scroll-margin-top: 80px; /* высота хедера */
}
```

## Задача 4 — Фильтр по ТЦ: оставить только три нужных

Найди фильтр по торговым центрам на сайте афиши (скорее всего это `<select>`, список кнопок или чекбоксов).

Оставь в фильтре **только эти три варианта**:
- Кунцево Плаза
- Каширская Плаза
- Капитолий

Удали все остальные пункты если они есть (например «Все ТЦ» можно оставить как опцию по умолчанию, остальные лишние — убрать).

Проверь:
- Если фильтр рендерится динамически из JS-массива или JSON — найди этот массив и очисти его, оставив только три нужных значения
- Если фильтр захардкожен в HTML — удали лишние `<option>` / кнопки напрямую
- После изменения фильтрация по каждому из трёх ТЦ должна работать корректно
- Значения (`value`) должны совпадать с тем что записано в данных мероприятий (например `kuntsevo`, `kashirskaya`, `kapitoliy`)
