# Техническая спецификация: 3 лендинга детских клубов + доработка сайта афиши

## Контекст проекта

Уже существует сайт-афиша мероприятий (HTML + CSS + JS, без фреймворка), на котором реализовано:
- Список мероприятий
- Страница отдельного мероприятия
- Фильтрация по ТЦ
- Кнопка перехода в мобильное приложение (App Store / Google Play)

Нужно сделать три блока работ:
1. **Три лендинга** детских клубов, которые ссылаются на существующий сайт афиши
2. **Доработка страницы мероприятия** на существующем сайте — добавить веб-форму бронирования билетов
3. **Доработка главной страницы афиши** — добавить секцию бронирования ивент-площадок с календарём занятости и формой заявки

---

## БЛОК 1 — Три лендинга детских клубов

### Общие требования

- Стек: **чистый HTML + CSS + JS** (без фреймворков, в соответствии с существующим проектом)
- Три отдельных HTML-файла (или три варианта конфигурации одного шаблона) — по одному на каждый ТЦ
- **Единая структура и шаблон** для всех трёх
- Визуальные отличия между ТЦ: только **цветовая схема** и **изображения**
- **Адаптивная вёрстка**: корректное отображение на десктопе и мобильных устройствах
- **Стилистика** берётся из существующего сайта афиши — переиспользовать CSS-переменные, типографику, компоненты

### Три ТЦ и их домены

| ТЦ | Домен лендинга | Основной сайт |
|----|----------------|---------------|
| Кунцево Плаза | kids.kuntsevoplaza.ru | kuntsevoplaza.ru |
| Каширская Плаза | kids.kashirskaya-plaza.ru | kashirskaya-plaza.ru |
| Капитолий | kids.kapitoliy.ru | kapitoliy.ru |

### Переключение между ТЦ

- В шапке или отдельном блоке — **переключатель / навигация между тремя ТЦ**
- При переключении меняется: цветовая схема, изображения, контактные данные, ссылки на приложение
- Реализация: либо три отдельных страницы со ссылками друг на друга, либо один шаблон с JS-переключением — выбрать наиболее поддерживаемый вариант

### Структура каждого лендинга (блоки сверху вниз)

#### 1. Шапка (Header)
- Логотип детского клуба / ТЦ
- Навигация по блокам страницы (якорные ссылки)
- Переключатель между тремя ТЦ

#### 2. Hero-блок (главный экран)
- Заголовок и краткое описание детского клуба
- Кнопка «Записаться» — ведёт в мобильное приложение:
  - App Store: ссылка на приложение конкретного ТЦ
  - Google Play: ссылка на приложение конкретного ТЦ
- Фоновое изображение или визуальный акцент

#### 3. Место проведения
- Название и адрес площадки
- Описание помещения
- Фотографии площадки
- Карта или схема проезда (опционально — iframe Google Maps или статичная ссылка)

#### 4. Афиша мероприятий
- Заголовок секции «Афиша» или «Расписание»
- Краткий анонс ближайших мероприятий (3–6 карточек) — данные можно брать из существующего сайта афиши или захардкодить как заглушки
- Кнопка «Смотреть все мероприятия» → ссылка на существующий сайт афиши, с предустановленным фильтром по данному ТЦ (например: `?tc=kuntsevo`)

#### 5. Галерея работ детей
- Сетка изображений (фотографии работ / занятий)
- Лайтбокс при клике на фото (открытие в полном размере)
- Placeholder-изображения если реальных нет

#### 6. Отзывы родителей
- Карточки с отзывами (имя родителя, текст, опционально — фото или инициалы)
- Минимум 3 карточки, можно со слайдером если их больше 4

#### 7. Контакты
- Адрес ТЦ
- Телефон
- Email
- Ссылки на соцсети (если есть)
- Кнопка «Скачать приложение» (App Store + Google Play) — повтор CTA

#### 8. Подвал (Footer)
- Копирайт
- Ссылка на основной сайт ТЦ
- Навигация между тремя ТЦ (дублирование переключателя)

### CSS-архитектура

- Использовать **CSS-переменные** для цветовой схемы, чтобы переключение между ТЦ менялось заменой одного набора переменных
- Пример структуры переменных:
```css
/* Кунцево Плаза */
:root {
  --color-primary: /* цвет из существующего сайта */;
  --color-accent: /* акцентный цвет */;
  --color-bg: /* фоновый цвет */;
}

/* Каширская Плаза */
.theme-kashirskaya {
  --color-primary: /* другой цвет */;
  --color-accent: /* другой акцент */;
}
```
- **Важно**: всю типографику, отступы, компоненты (кнопки, карточки, хедер) взять из существующего сайта афиши — не изобретать заново

### Файловая структура (предлагаемая)

```
/kids-landings/
  index.html              ← редирект или страница выбора ТЦ
  kuntsevo/
    index.html
  kashirskaya/
    index.html
  kapitoliy/
    index.html
  css/
    base.css              ← общие стили (из существующего проекта)
    landings.css          ← стили лендингов
    themes.css            ← цветовые схемы по ТЦ
  js/
    landings.js           ← галерея, слайдер отзывов, переключатель ТЦ
  images/
    kuntsevo/
    kashirskaya/
    kapitoliy/
```

## БЛОК 2 — Доработка страницы мероприятия на сайте афиши

### Где добавляется

На **странице отдельного мероприятия** (уже существующей), в самом низу — после всего остального контента страницы.

### Что добавить

Два элемента последовательно:

#### 1. Блок «Купить билет / Записаться»
- Кнопки App Store и Google Play с иконками
- Краткий поясняющий текст (например: «Запись на мероприятие — через мобильное приложение»)

#### 2. Веб-форма бронирования

Заголовок: «Или оставьте заявку»

Поля формы:
- **Имя** (text, обязательное)
- **Телефон** (tel, обязательное)
- **Email** (email, обязательное)
- **Количество человек** (number, обязательное)
- **Комментарий** (textarea, необязательное)
- Скрытые поля, автоматически заполняемые из контекста страницы:
  - Название мероприятия
  - Дата мероприятия
  - ТЦ / площадка

Кнопка: «Отправить заявку»

### Отправка формы

- **Метод**: `fetch` POST-запрос на серверный endpoint (или mailto-fallback если бэкенда нет)
- **Что уходит на почту сотрудников**: все поля формы + автоматически подставленные данные мероприятия
- Реализовать **серверную часть** (Node.js / PHP / любой доступный в проекте способ) для отправки письма через SMTP или через сервис (например, Nodemailer, SendGrid, FormSubmit.co как fallback)
- **Email получателя**: задаётся в конфиге / переменной окружения (`BOOKING_EMAIL=...`), не хардкодится в коде
- После успешной отправки: показать сообщение «Заявка принята, мы свяжемся с вами» без перезагрузки страницы
- При ошибке: показать сообщение об ошибке, не терять введённые данные

### Валидация

- Клиентская: HTML5-атрибуты + JS-валидация перед отправкой
- Обязательные поля подсвечиваются при попытке отправить пустыми
- Телефон: базовая проверка формата

### Стилизация формы

- Использовать **существующие CSS-стили** из проекта (цвета кнопок, инпутов, типографика)
- Форма должна органично вписываться в страницу мероприятия, не выглядеть чужеродно

## БЛОК 3 — Доработка главной страницы афиши: секция бронирования ивент-площадок

### Где добавляется

В самом низу **главной страницы сайта афиши** — после списка мероприятий, отдельной секцией с якорем `#venue-booking`.

---

### 3.1 — Переключатель ТЦ

- В начале секции — **выпадающий список (select / dropdown)** для выбора ТЦ:
  - Кунцево Плаза
  - Каширская Плаза
  - Капитолий
- При смене ТЦ — обновляются: календарь, список залов в форме, email получателя заявки
- Выбранный ТЦ сохраняется в URL-параметре (`?venue=kuntsevo`) чтобы можно было дать прямую ссылку

---

### 3.2 — Календарь занятости

#### Внешний вид
- Стандартный месячный календарь (сетка пн–вс)
- Навигация: кнопки «предыдущий месяц» / «следующий месяц»
- Каждый день — кликабельная ячейка

#### Цветовая индикация дней
- 🟢 **Зелёный** — день полностью свободен
- 🟡 **Жёлтый** — день частично занят (есть занятые слоты)
- 🔴 **Красный** — день полностью занят
- ⬜ **Серый** — прошедшая дата (не кликабельна)

#### Попап при клике на день
При клике на любой не прошедший день открывается **модальное окно / попап**:
- Заголовок: дата (например, «Суббота, 14 июня»)
- Список залов выбранного ТЦ, у каждого — временные слоты (например, 10:00–13:00, 13:00–17:00, 17:00–22:00)
- Каждый слот окрашен:
  - 🟢 Зелёный фон — свободен
  - 🔴 Красный фон — занят
- Кнопка «Забронировать этот день» — закрывает попап и автоматически заполняет поле даты в форме ниже
- Кнопка закрытия попапа (×)

#### Источник данных
- Данные о занятости хранятся в **JSON-файле** (`/data/venue-schedule.json`) — статичный на старте, обновляется вручную
- Структура JSON:
```json
{
  "kuntsevo": {
    "halls": ["Зал 1 этаж", "Зал 2 этаж"],
    "schedule": {
      "2026-06-14": {
        "Зал 1 этаж": {
          "10:00–14:00": "busy",
          "14:00–18:00": "free",
          "18:00–22:00": "free"
        },
        "Зал 2 этаж": {
          "10:00–14:00": "free",
          "14:00–18:00": "busy",
          "18:00–22:00": "busy"
        }
      }
    }
  },
  "kashirskaya": { ... },
  "kapitoliy": { ... }
}
```
- **Важно**: JSON-файл должен быть легко редактируемым вручную — это временное решение до появления админки

---

### 3.3 — Форма заявки на бронирование площадки

Форма расположена **сразу под календарём**, в той же секции.

#### Заголовок
«Оставить заявку на бронирование площадки»

#### Поля формы

| Поле | Тип | Обязательное |
|------|-----|--------------|
| Название мероприятия | text | ✅ |
| Описание мероприятия | textarea | ✅ |
| Желаемая дата | date (или text с маской) | ✅ (автозаполняется из календаря) |
| Время начала — время окончания | два поля time | ✅ |
| Выбор зала / площадки | select — список залов выбранного ТЦ | ✅ |
| Контактное лицо (имя) | text | ✅ |
| Телефон | tel | ✅ |
| Email | email | ✅ |
| Загрузка баннера мероприятия | file (jpg/png, до 5 МБ) | ❌ |
| Загрузка презентации | file (pdf/pptx, до 20 МБ) | ❌ |
| Комментарий | textarea | ❌ |

Скрытое поле (автоподставляется): выбранный ТЦ

#### Кнопка
«Отправить заявку»

#### Отправка
- `fetch` POST на серверный endpoint
- На почту сотрудников уходит письмо со всеми полями + прикреплёнными файлами (если загружены)
- **Email получателя** задаётся в конфиге по каждому ТЦ отдельно:
```js
const VENUE_EMAILS = {
  kuntsevo: process.env.EMAIL_KUNTSEVO,
  kashirskaya: process.env.EMAIL_KASHIRSKAYA,
  kapitoliy: process.env.EMAIL_KAPITOLIY
};
```
- После успешной отправки: сообщение «Заявка принята, мы свяжемся с вами в ближайшее время» без перезагрузки страницы
- При ошибке: сообщение об ошибке, данные формы не теряются

#### Валидация
- Обязательные поля — HTML5 + JS-проверка перед отправкой
- Телефон — базовый формат
- Файлы — проверка типа и размера на клиенте перед отправкой

---

### 3.4 — UX-связка календаря и формы

- Клик на день в календаре → попап со слотами → кнопка «Забронировать» →
  - Попап закрывается
  - Страница плавно скроллится к форме (`scroll into view`)
  - Поле «Желаемая дата» автоматически заполняется выбранной датой
  - Поле «Зал» автоматически выставляется если был выбран конкретный зал в попапе
  - Поля подсвечиваются (анимация highlight) чтобы пользователь видел что они заполнились
